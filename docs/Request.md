# BitMart

## 背景

目前我校跳蚤市场建立在 QQ 群上。在群里，卖家通常都是将商品拍照后附价格，而附文字描述较少，对于二手书尤甚。卖方这样很方便，但买方就比较麻烦，需要一张张图片找，而且可能找到别人已经售出的商品。

因此我想开发这样一个跳蚤市场应用：卖方可以扫描书本条形码并填写售价、书况等信息，也可以对一本或多本书封面或书脊拍照后调用 OCR 或 LLM 批量添加，然后将信息上传到服务器数据库。对其他商品也同样可以提供拍照识别等功能，填写信息后上传到服务器。买方则可以搜索自己需要的商品（兼顾商品名、商品描述等字段（有可能中英文混合），最好能模糊搜索，允许设置期望价格等），并获取卖方联系方式。商品可以添加 tag 并允许通过标签筛选。同一条出售单可能包含多件同类商品并且逐个售卖。

## 功能设计

### 号

用户注册时必须提供统一身份认证系统的学号和密码。调用 BIT101 服务器提供的 API，在本地对密码加密后进行验证，验证通过的才允许注册，要求用户设置 BitMart 账号的密码，提交给服务器并加密保存。

登录时提供学号和 BitMart 密码即可，忘记密码时利用统一身份认证系统验证身份后重新设置密码。

允许用户设置昵称（不设置即显示为匿名）。

可以查看来自服务端的通知，例如公告、超时提醒等。

必须登录才可以发布买卖请求。

注销时，级联删除该用户发布的所有商品、需求。

### 卖

填写商品的名称、件数、单价（允许留空，即买方带价联系或双方联系上以后再议价）、标签（设置个数上限）、描述、图片、联系方式（必填）、取货地点（例如某某宿舍楼）、过期时间（设置最少、最多、默认多少天后过期）等信息。本地存储常用联系方式。允许用户填写多件商品后批量上传。

特别地，对于书籍，还可以填写 ISBN、作者、出版社、版本等信息。可以本地批量扫描条形码获取 ISBN，向服务器提交请求，根据 ISBN 查询书籍信息。用户可以对查询到的信息作调整修改。服务器可以调用 API 根据 ISBN 查询书籍信息，并将结果存储在服务器，未来面对相同的请求不必调用 API。

配置 LLM API，用于本地调用 API 识图。设置默认提示词，允许用户自行调整。同时用 OpenAI Compatible 协议的 `response_format` 参数和提示词约束输出格式。LLM 识别图中信息并按格式返回后，解析出的商品信息用户同样可以进行调整修改。要求用户在拍照给 LLM 之前选择是否为书籍。

用户可以调整商品信息，以及调整售出数量。当用户删除商品时，我们应当仅将数据标记为删除而非物理删除。

临近超时时通知提醒用户，允许用户延期（同样最少 1 天、最多 365 天、默认 30 天后）。

### 买

填写求购需求，格式与商品格式相同，唯此时价格表示期望价格。同样设置超时提醒。

### 商品列表

展示商品，允许按时间排序、按标签筛选、按价格区间筛选（允许用户选择是否列出价格为空的商品）、用文字搜索。允许用户选择是否展示售罄商品。详情页显示卖家昵称、联系方式。不显示过期的商品。

不对未登录的客户端显示详情。

### 需求列表

类似商品列表。

## 说明

### 安全与隐私

不实现任何交易功能，要求用户自行联系、协商、交易，警告用户验明身份、小心诈骗。

鼓励用户使用微信、QQ 进行联系，警告使用手机号带来的隐私风险。

### 代码风格

充分使用常量、配置文件。

按照官方规范充分编写注释，但务必简洁明了，避免无意义的过度注释，以及用序号编号各步骤。

### 技术路线

采用测试驱动开发模式，充分编写测试用例，充分考虑边界情况。

规范记录日志。

各个第三方库均应使用最新稳定版，采用推荐实践，避免使用 deprecated 功能。

考虑到我们的规模不大，可以选用相对轻量的方案，未来有需要再进行升级优化。

后端采用 Kotlin 开发。

Android 端采用 Jetpack Compose。采用 Clean Architecture。

注意到买和卖几乎是对偶的，可以考虑用同一套结构实现。

## 部署

我拥有自己的内网 Linux 服务器。我朋友有带公网 IP 和域名的 Linux 服务器，且我的 Tailscale 账号和他的组了网络。既可以让他将转发请求到我的服务器，也可以直接将服务部署在他的服务器上。我们先在本地调试，经测试没有问题再上线部署。

## 改进方向

### 显示

支持多语言。

支持亮暗两主题和自定义颜色方案。

### 频率限制

对用户调用接口进行频率限制。注销后设置窗口期，禁止频繁注销注册。

### 权限管理

区分管理员和普通用户，管理员可以删除商品和收购需求，也可以封禁用户。

### NapCat 导入商品信息

若征得现有跳蚤市场管理员同意，可以向跳蚤市场群接入 NapCat 机器人，接入 LLM 进行商品导入。此时联系方式仅有 QQ，且身份未经验证，提醒用户风险即可。
